Exam Day Playbook

Rohan-specific strategy — you need 14 more points. Here's exactly how to get them.

🎯 High-yield topics from the official outline
📋
ARM Templates & Bicep (officially on exam): Know the 5 required sections of an ARM template: $schema, contentVersion, resources (required) + parameters, variables, outputs, functions (optional). Bicep uses param, var, output keywords. You may be asked to read a snippet and identify what it does.
🔄
Deployment Slots (App Service): Swapping slots swaps ALL settings by default EXCEPT slot-specific settings (connection strings / app settings marked as "slot setting"). Traffic splitting uses slot percentages. Blue/green = swap with no downtime. This is a common scenario question.
🗑️
Soft Delete (NEW in April 2025 update): Soft delete is now explicitly listed for BOTH blobs AND containers. When enabled, deleted blobs/containers are retained for a configurable retention period. Restoring = undelete operation. Separate settings for blobs vs containers.
🔐
Service Endpoints vs Private Endpoints: Service Endpoint = traffic stays on Azure backbone but storage account is still reachable via public IP (just restricted to your VNet). Private Endpoint = storage account gets a private IP in your VNet — fully private. Private Endpoint is more secure and works across peered VNets.
📦
Azure Container Registry (ACR) — now explicitly on exam: ACR stores Docker images. To use ACR with ACI or Container Apps, you grant the AcrPull role to the managed identity. ACR tiers: Basic, Standard, Premium (geo-replication = Premium only). You push with docker push or az acr build.
💰
Cost Management (officially listed): Azure Cost Management has Budgets (set spending limits + alerts), Cost Alerts (budget/credit/department quota alerts), and Azure Advisor Cost recommendations (right-size VMs, delete unused resources, reserved instance suggestions).
🏷️
Tags — scope and inheritance: Tags are NOT inherited by default (resources in a tagged RG don't automatically get the tag). You must use Azure Policy (Inherit a tag from resource group) to force inheritance. ARM templates CAN assign tags. Management groups CANNOT have tags.
🔒
Resource Locks — Read-Only vs Delete: ReadOnly lock = no changes AND no new resources (even reads that trigger side effects like listing keys are blocked). Delete lock = can modify, cannot delete. Locks apply to ALL users including Owner role. Must remove lock to delete.
